# Chinese Field Guide

A personal HTML/CSS/JS field guide to Chinese language and civilization. Characters, vocabulary, grammar, religion, philosophy, history, geography, culture, culinary traditions, arts and literature, science and medicine, and everyday life.

Static site generated by a Node build system. Source of truth is `content/`. Never hand-edit `pages/` — they are generated.

---

## For AI Agents — Read This First

This repo is set up for incremental authoring across many sessions. The key rules:

1. **Never hand-edit `pages/` or `data/`.** All pages are generated by `npm run build` from `content/<category>/<slug>.md`. Adding a page means: create the markdown file, then run the build.
2. **All content files live under `content/<category>/`** — one `.md` per page. Generated pages land in `pages/[category]/`.
3. **Flip `status` in the frontmatter** from `stub` to `complete` when a page is fully authored, and set `updated` to today's date.
4. **HSK pages are deferred** — `pages/hsk/` data exists but no HTML pages. Do not create HSK content without being explicitly asked.
5. **Full project conventions are in `CLAUDE.md`** — read it before creating or editing any content page.

Reference exemplars: `content/characters/dao4_道.md` (character), `content/philosophy/topic_kongzi.md` (topic), `content/grammar/ba_把.md` (grammar), `content/vocab/guanxi_关系.md` (vocab).

## File Conventions

### Path depth
Generated pages are at `pages/[category]/filename.html` — two levels from root:
- Stylesheet: `../../style.css`
- Index: `../../index.html`
- Shared script: `../../scripts/toc-scroll.js`

### Stylesheet
Always `style.css` at the repo root. Never rename it.

### Shared script
`scripts/toc-scroll.js` handles TOC scroll-spy and mobile sidebar toggle. It is injected by `_layout.html` at build time — do not add it manually to content `.md` files.

### Naming
- Character pages: `[pinyin][tone]_[char].md` → `content/characters/gan3_感.md`
- Vocab pages: `[pinyin-ascii]_[char].md` → `content/vocab/mianzi_面子.md`
- Grammar pages: short ASCII slug → `content/grammar/le_了.md`
- Topic pages: `topic_[slug].md` → `content/religion/topic_chan.md`

ASCII-only filenames — no toned vowels (ā á ǎ à etc.) in filenames.

### Adding a new page
1. Create `content/<category>/<slug>.md` with valid YAML frontmatter.
2. Run `npm run validate` to check the schema.
3. Run `npm run build` to generate `pages/<category>/<slug>.html`.

### Authoring queue
The full entry checklist lives in `local/authoring-queue.md` (gitignored — local only).
